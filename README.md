# 🍱 Gourmet Lunch Finder (ランチアプリ)

[![Live Demo](https://img.shields.io/badge/Demo-Live%20Now-success?style=for-the-badge)](https://oh-zh-ynu.github.io/lunch-app/)

毎日のランチタイムにぴったりのお店を検索し、気になるお店をストックできるモダンなWebアプリケーションです。
サーバーサイドの構築は不要で、安全に外部APIと連携する完全な静的クライアントサイドアプリとして実装されています。

## ✨ 主な機能

1. **ランチ特化の検索とフィルタリング**
   - 「場所」と「条件（ジャンル、キーワードなど）」を組み合わせた柔軟な検索が可能。
   - ディナー予算しか返さない標準APIを補強し、フロントエンドでの**ランチ予算による絞り込み**を独自実装。
   - 個室、Wi-Fi、飲み放題などの「こだわり条件」フィルターに対応。

2. **店舗情報の自動拡張（Webスクレイピング）**
   - APIから取得できない「正確なランチ予算」と店舗ページの「最新の更新日」を、`IntersectionObserver`を用いてバックグラウンドで非同期に遅延読み込み（スクレイピング）します。
   - 抽出したデータはカードの右下等の最適な位置に動的にマージされます。

3. **ストック＆パーソナルメモ機能**
   - 気になるお店をワンクリックで一覧に保存（ストック）できます。
   - お店ごとに「平日限定ランチがおすすめ」などの自分専用のメモを残せます。
   - データはユーザーの端末（ブラウザの `localStorage`）に安全に保存されます。

4. **デュアルプロキシによる堅牢なAPI通信**
   - GitHub Pagesのような静的ホスティング環境からCORS制約を回避しつつ安定して外部APIを叩くため、`corsproxy.io` と `allorigins.win`、`codetabs.com` などの複数の中継サーバーを自動でフォールバック（切り替え）する通信ロジックを実装しています。

5. **洗練されたUI/UX**
   - Glassmorphism（ガラス風デザイン）とモダンなアニメーションを採用した、直感的で心地よいインターフェース。
   - スマホ・タブレット・PCのすべてで最適に表示されるレスポンシブデザイン。

## �️ 技術スタック

- **フロントエンド**: HTML5, Vanilla JavaScript (ES6+), CSS3
- **デザイン**: Glassmorphism, カスタムCSS変数, [Phosphor Icons](https://phosphoricons.com/)
- **API**: [HotPepper グルメ Web サービス API](https://webservice.recruit.co.jp/hotpepper/)
- **インフラ**: GitHub Pages (Hosting)

## 📁 主要ファイル構成

- `index.html`: アプリの骨組みとなるメインのHTMLファイル
- `styles.css`: Glassmorphismやアニメーション、レイアウトを定義するCSS
- `app.js`: 検索ロジック、エリア取得、スクレイピング、状態管理(localStorage)を行うJavaScript

## ⚠️ 注意事項・クレジット

- 本アプリは「HotPepper グルメ Web サービス API」を利用してデータを取得していますが、公式のアプリケーションではありません。
- 検索画面下部には、API利用規約に則り公式のクレジット表記（ロゴ）を配置しています。
- スクレイピング機能はサーバーに過度な負荷をかけないよう、同時実行数を制限するキューシステム（遅延実行）を設けて実装しています。
